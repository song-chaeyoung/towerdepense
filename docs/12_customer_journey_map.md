# Step 12. 고객 여정 맵

> 목적: 팀이 **우리 서비스**에서 사용자가 **어떤 순서로 무엇을 하는지** 같은 그림으로 이해한다. (Step 05는 기존·대체 방식, Step 12는 우리 서비스)

> **Guide:**  
> 1. Agent가 `docs/11_page_routing_split.md`만 읽고 **고객 여정 맵 초안**을 작성한다.  
> 2. 팀이 표를 보며 단계·행동·감정이 맞는지 이야기하고 수정한다.  
> - **데모에서 보여줄 핵심 여정 1개**를 중심으로 쓴다.  
> - Step 11의 **페이지·기능·이동 흐름**을 바탕으로 단계를 구성한다.  
> - 각 단계는 **구체적인 행동**으로 쓴다. (예: "앱을 연다" → "일정을 입력하고 추천 버튼을 누른다")  
> - Step 05처럼 **표 한 벌**로 정리한다. 팀원 모두가 흐름을 따라갈 수 있어야 한다.

**프롬프트 예시**

```
docs/11_page_routing_split.md만 읽고 docs/12_customer_journey_map.md를 채워줘.

규칙:
- Step 11의 페이지 구성·기능 목록·페이지 간 이동 흐름만 참고
- 우리 서비스 기준의 핵심 여정 1개 (데모 가능한 흐름)
- 단계 | 사용자 행동 | 생각/감정 | 해당 페이지 | 사용 기능(F-xx) 표 작성
- Step 11에 있는 페이지와 F-xx를 빠짐없이 연결
- 구조와 상단 Guide는 유지
```

## 여정 요약

> **누가** → **무엇을 하려고** → **어떤 순서로** → **무엇을 얻는다**

- **잠깐 즐길 게임을 찾던 사용자**가 → **부담 없이 한 판 하려고** → **링크 접속 → 시작 → 타워 배치·웨이브 방어 → 결과 확인** 순으로 진행해 → **몇 분 만에 클리어(또는 재도전)의 성취와 점수**를 얻는다.

## 고객 여정

> MVP·데모에서 보여줄 주 사용자 흐름

| 단계 | 사용자 행동 | 생각/감정 | 해당 페이지 | 사용 기능 |
|---|---|---|---|---|
| 1. 접속 | 공유받은 링크를 열자 타이틀·규칙 한 줄·최고기록이 즉시 뜬다 | "오, 바로 뜨네. 뭐 하는 게임인지 알겠다" | 메인(MENU) | F-01, F-25, F-30 |
| 2. 시작 | [게임 시작]을 눌러 맵으로 들어간다 | "가볍게 한 판 해보자" | 메인→게임 | F-01 |
| 3. 배치 | 타워 종류를 고르고(사거리 원 확인) 빈 타일에 설치한다 | "여기 길목에 세우면 잘 막겠지?" | 게임(PLAYING) | F-04~F-09, F-24 |
| 4. 방어 | "웨이브 시작"을 누르고 몰려오는 적을 타워로 처치한다 | "오 잘 잡힌다. 골드 벌었다!" | 게임(PLAYING) | F-10~F-18, F-23 |
| 5. 운영 | 번 골드로 타워를 추가·강화(필요 시 판매)하며 다음 웨이브 대비 | "지금 강화할까, 새 타워 지을까?" | 게임(PLAYING) | F-19~F-22, F-16, F-20 |
| 6. 결과 | 목숨 0(패배) 또는 전 웨이브 방어(승리) 시 결과 요약을 본다 | "아쉽다 다시!" / "클리어했다!" | 결과(RESULT) | F-26~F-28, F-30 |
| 7. 재도전·공유 | [다시 하기]로 재시작하거나 결과 링크를 친구에게 공유한다 | "한 번 더" / "이거 너도 해봐" | 결과(RESULT) | F-29, F-31 |
