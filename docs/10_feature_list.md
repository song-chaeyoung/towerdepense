# Step 10. 기능 나열

> 목적: 지금까지 논의에서 나온 **기능 아이디어를 하나도 빠뜨리지 않고** 모은다.

> **Guide:**  
> 1. Agent가 `docs/04`~`docs/09`을 읽고, 언급된 **모든 기능**을 추출한다.  
> 2. **순서·중요도·선후 관계 없이** 일렬로 나열한다.  
> 3. 팀이 빠진 기능이 있으면 **팀이 추가한 기능**에 적는다.  
> - 기능은 **사용자가 할 수 있는 일** 또는 **서비스가 제공하는 것** 단위로 쓴다. (예: 로그인, 일정 자동 추천, 알림 보내기)  
> - 큰 덩어리(예: "앱 전체")로만 적혀 있어도 **일단 포함**하고, 안에 여러 기능이 섞여 있으면 **풀어서** 각각 적는다.  
> - 비슷한 기능이라도 **논의에서 따로 나왔으면 각각** 적는다.  
> - **이 단계에서 하지 않는 것:** 우선순위 정하기, 중요도 판단, MVP만 골라내기, 제외·합치기  
> - 우선순위·제외는 **Step 14**에서 한다. 여기서는 **놓치지 않고 모으는 것**만 한다.

**프롬프트 예시**

```
docs/04~09을 읽고 docs/10_feature_list.md의 "Agent가 추출한 기능" 섹션만 채워줘.

규칙:
- docs/04~09 어디에든 나온 기능·아이디어를 빠짐없이 포함 (목적·가치, 여정, Pain/Wow, 가설, 키워드, 질문·답변 등)
- F-01부터 번호를 매겨 일렬로 나열 (번호·순서는 우선순위가 아님)
- 기능명은 짧고 구체적으로 (한 줄)
- 우선순위 판단·제외·MVP 선별 하지 않는다. 의심스러워도 일단 나열
- "팀이 추가한 기능" 섹션은 건드리지 않는다.
- 구조와 상단 Guide는 유지
```

## Agent가 추출한 기능

> `docs/04`~`docs/09`에서 나온 기능. **빠짐없이** 나열. Agent가 채운다.

- **F-01** 접속 시 로딩·인트로 없이 게임 맵이 바로 뜨는 즉시 플레이 진입
- **F-02** 격자 기반 게임 맵 렌더링 (Canvas)
- **F-03** 적이 지나가는 고정 경로 표시 (도로색)
- **F-04** 설치 가능한 빈 타일 표시 (잔디색 + 격자선)
- **F-05** 빈 타일 클릭으로 타워 설치
- **F-06** 타워 종류 선택 UI (이름·비용·특성 표시)
- **F-07** 타워 선택 시 사거리(range) 미리보기 원 표시
- **F-08** 마우스 호버 타일에 설치 가능/불가 하이라이트
- **F-09** 여러 종류의 타워 제공 (사거리/연사/광역/둔화 등 성격 차이)
- **F-10** 타워 자동 조준·발사 (사거리 내 적 공격)
- **F-11** 발사체(projectile) 이동·명중 처리
- **F-12** 적 유닛 스폰 및 경로 따라 이동
- **F-13** 여러 종류의 적 제공 (일반/빠른 적/탱커 등)
- **F-14** 적 체력(HP) 및 체력바 표시
- **F-15** 타워 공격으로 적에게 피해 적용·처치
- **F-16** 웨이브 시스템 (웨이브 시작 버튼, 순차 스폰)
- **F-17** 웨이브별 적 구성·난이도 변화
- **F-18** 골드 시스템 (적 처치 시 골드 획득)
- **F-19** 골드로 타워 설치·강화 비용 지불
- **F-20** 웨이브 클리어 시 보너스 골드 지급
- **F-21** 타워 강화(업그레이드)
- **F-22** 타워 판매(환불)
- **F-23** 목숨 시스템 (적이 끝에 도달하면 목숨 감소)
- **F-24** 상단 HUD (골드·목숨·현재 웨이브 표시)
- **F-25** 첫 진입 시 조작 힌트 한 줄 표시
- **F-26** 게임오버 처리 (목숨 0)
- **F-27** 승리 처리 (전 웨이브 방어)
- **F-28** 결과 요약 (도달 웨이브·처치 수·남은 목숨)

- **F-30** 점수/도달 웨이브 기록 (하이스코어 확장 여지)
